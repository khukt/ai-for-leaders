Prompting and workflow design

The model stays general-purpose, but instructions, guardrails, and process design shape how it behaves in use.

Result Workflow assistant

Fastest route to a usable system for drafting, summarising, support, and low-risk productivity work.

Use the model as-is, but wrap it in prompts, decision rules, templates, and human review.

Best fit
Drafting, summarisation, search assistance, and workflow copilots.
Cost profile
Lowest upfront cost and fastest time to trial.
Leadership watchpoint
Local productivity gains can be mistaken for strategic capability if the workflow itself does not change.

Add trusted internal documents or knowledge sources at run time so the model answers with current organisational context.

Best fit
Knowledge assistants, policy search, research support, and controlled enterprise Q&A.
Cost profile
Moderate build cost because retrieval quality, permissions, and source freshness become critical.
Leadership watchpoint
Weak data stewardship or poor source quality will undermine trust faster than model quality alone.

Change the model itself so it performs better in a narrow domain, language, or task pattern.

Best fit
Domain-heavy tasks where general prompts are not enough and the advantage is expected to persist.
Cost profile
Higher cost because data quality, evaluation, and operating discipline all matter more.
Leadership watchpoint
Only worth it when the advantage is durable, not when a general model is likely to catch up quickly.

Tune the system's responses around safety, refusal, tone, escalation, and other policy-sensitive behaviours.

Best fit
Public-facing assistants, sensitive workflows, and systems operating under stronger trust or safety expectations.
Cost profile
Moderate to high because testing edge cases and maintaining policy consistency takes ongoing work.
Leadership watchpoint
Alignment is not only a model setting; it becomes part of governance, complaints handling, and accountability.

Reduce model size or deployment weight so it can run faster, cheaper, or closer to the edge.

Best fit
Latency-sensitive, resource-constrained, on-device, or edge deployments.
Cost profile
Can reduce operating cost later, but requires careful testing to avoid quality loss.
Leadership watchpoint
The trade-off is usually between efficiency and capability, not between efficiency and zero risk.

Figure: many organisations start with a general-purpose foundation model and then adapt, align, or constrain it for narrower business uses rather than building from scratch.
